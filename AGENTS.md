# AGENTS.md

## Project Overview

Hexlet educational project with Django backend and Svelte frontend.

- **Backend**: Django with PDM for dependency management
- **Frontend**: Svelte 5 + Vite with Docker deployment

---

## Quick Start with Make

A `Makefile` is provided for convenient command execution. Run `make` or `make help` to see available commands.

### Common Make Commands

```bash
# Installation
make install           # Install all dependencies
make backend-install   # Install only backend dependencies
make frontend-install  # Install only frontend dependencies

# Docker operations
make up                # Start all services via Docker Compose
make up-build          # Build and start all services
make down              # Stop all Docker services
make restart           # Restart all services
make logs              # View backend logs
make logs-frontend     # View frontend logs
make status            # Check containers status

# Local development
make backend-dev       # Start backend dev server (local)
make frontend-dev      # Start frontend dev server (local)
make migrate           # Apply Django migrations (local)
make makemigrations    # Create Django migrations (local)
make createsuperuser   # Create admin user (local)
make test              # Run Django tests (local)

# Maintenance
make clean             # Clean up cache and artifacts
make help              # Show all available commands
```

## Backend (PDM + Django)

**All commands must run from `backend/` directory.**

### Setup

```bash
cd backend
pdm install
cp .env.example .env  # Edit SECRET_KEY before production
pdm run migrate
```

Or use Make:
```bash
make backend-install
```

### Key Commands

| Command | Purpose |
|---------|---------|
| `pdm run dev` | Start development server |
| `pdm run migrate` | Apply migrations |
| `pdm run makemigrations` | Create migrations |
| `pdm run test` | Run tests |
| `pdm run shell` | Django shell |
| `pdm run createsuperuser` | Create admin user |

### Docker Deployment

**All commands must run from project root.**

```bash
# Build and start all services (using Make)
make up-build

# Or using docker-compose directly
docker-compose up --build -d

# View backend logs
make logs
# Or
docker-compose logs -f backend

# Stop all services
make down
# Or
docker-compose down
```

Access:
- Backend: http://localhost:8000
- Django Admin: http://localhost:8000/admin/

### Backend Docker Structure

```
backend/
├── Dockerfile            # Python 3.12 + PDM
├── entrypoint.sh         # Entrypoint with migrations
├── pyproject.toml        # PDM dependencies
└── .env                  # Environment variables
```

### Critical Conventions

- **Python 3.12 strictly required** (`requires-python = "==3.12.*"`)
- **PDM only** — never use pip directly
- **Settings module**: `server.settings`
- **Environment variables**: Loaded via `python-dotenv` from `.env`
- **Django 6.0.4** — check docs at django.readthedocs.io/en/6.0

### Creating Apps

```bash
pdm run python manage.py startapp <app_name>
```

Add to `INSTALLED_APPS` in `server/settings.py`.

## Frontend (Svelte + Vite)

**All commands must run from `frontend/` directory.**

### Technology Stack

- **Svelte 5** — Компилируемый фреймворк UI
- **Vite 6** — Сборщик с мгновенным HMR
- **Node.js 20** — LTS версия для production (Alpine)
- **Nginx** — Веб-сервер для раздачи статики

### Setup

```bash
cd frontend
npm install
```

Or use Make:
```bash
make frontend-install
```

### Key Commands

| Command | Purpose |
|---------|---------|
| `npm run dev` | Start development server (port 5173) |
| `npm run build` | Build for production |
| `npm run preview` | Preview production build |

### Docker Deployment

```bash
# Build and run all services
make up-build

# View logs
make logs
# Or
docker-compose logs -f

# Stop
make down
# Or
docker-compose down
```

Access: http://localhost:8080

### Project Structure

```
frontend/
├── Dockerfile            # Multi-stage build (Node 20 + Nginx)
├── nginx.conf            # Nginx SPA configuration
├── vite.config.js        # Vite configuration
├── package.json          # Dependencies
└── src/
    ├── main.js           # Entry point
    ├── App.svelte        # Root component
    └── lib/              # Components
```

### Svelte 5 Runes

Use modern reactive syntax:

```svelte
<script>

  const increment = () => count += 1

```
